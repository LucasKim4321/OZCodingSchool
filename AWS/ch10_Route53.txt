Route53
AWS 서비스들과 쉽게 연동하여 사용할 수 있는 DNS 서비스.
EC2 ELB S3 CloudFront 등과 연동가능


* 호스팅 영역 *

호스팅 하고자 하는 영역을 지정
퍼블릭 호스팅 영역과 프라이빗 호스팅 영역이 있음.

퍼블릭 호스팅 영역
인터넷에서 트래픽을 라우팅하고자 하는 방법을 지정하는 레코드를 포함하는 영역

프라이빗 호스팅 영역
VPC에서 트래픽을 라우팅하고자 하는 방법을 지정하는 레코드를 포함하는 영역

레코드
도메인과 연결된 IP주소로 구성된 정보를 의미

DNS(domain Name System)
도메인 이름을 IP 주소로 바꿔주는 시스템
각각의 고유한 도메인 이름을 고유한 IP주소에 연결

DNS 쿼리
도메인 이름을 IP주소로 변환하는 요청


* 레코드 유형 *

A레코드 (Adress Record)
도메인 이름과 IPv4주소를 매핑하는 레코드

AAAA 레코드
도메인 이름과 IPv6주소를 매핑하는 레코드

CNAME(Canonical Name) 별칭 레코드
현재 레코드의 이름에 대한 DNS쿼리를 다른 도메인 또는 하위 도메인으로 매핑

MX 레코드 (Mail eXchange)
이메일을 전달하기 위한 레코드

NS 레코드 (Name Server)
네임 서버 주소를 담고 있는 레코드

SOA 레코드 (Start of Authority) 권한 시작 레코드
도메인에 대한 기본 DNS 정보 식별을 위한 레코드

NS레코드와 SOA레코드 두 가지는 호스팅 영역을 생성할 대 자동으로 생성되며,
함부로 변경하면 안됨!!


* 라우팅 방식 *

지연 시간 기반 라우팅 (Latency based Routing)
지연 시간(latency)이 가장 낮은 리전의 IP주소로 라우팅하는 방식

가중치 기반 라우팅(Weighted based Routing)
IP 주소나 ELB DNS주소에 각각 가중치(weight)를 부여하여 가중치에 따라서 라우팅하는 방식

지역 기반 라우팅(Geo Routing)
지역에 따라서 각기 다른 IP주소로 라우팅하는 방식
