** S3 (Simple Storage Service) **
저장 서비스
서버에서 각각의 저장소를 사용하게 되면 로드밸런싱할 때 데이터가 분산되기 때문에 하나의 저장소를 사용하기 위해 사용
영상 https://youtu.be/77lMCiiMilo

* 특징 *
객체 기반의 무제한 파일 저장 스토리지
URL을 통해 손쉽게 파일 공유 가능
99.9999999%의 내구성! 내구성이 좋다!
사용한 만큼만 지불 (GB당 과금)
정적 웹사이트 호스팅 가능

최종 파일을 저장하는 용도로 적합 (객체 스토리지의 특성상 파일의 일부분만 수정이 불가능하기 때문)

* 관련 용어 *

*버킷(Bucket)
객체를 담는 최상위 단위 (이름, 개수 제한 있음)

*객체(Object)
Bucket에 담는 데이터의 단위 (파일)

*메타데이터(Metadata)
Object에 대한 여러가지 정보를 담고 있는 데이터

*Policy(정책)
Bucket, Object에 대한 접근을 통제하는 권한 정보를

*S3 스토리지 클래스
스토리지를 저장하는 형태를 의미

Standard, Standard-IA, Glacier Flexible Retrieval 등이 있음

사용예)
Standard 30일 경과-> Standard-IA 30일 경과-> Glacier Flexible Retrieval

유지 비용이 높음에서 낮아짐
접근 비용이 낮음에서 높아짐

접근 빈도에 따라 클래스를 다르게 설정해 최대 82% 저렴하게 이용가능


*객체 스토리지(Object Storage)
객체 단위로 저장하는 스토리지
하나의 객체는 아이디와 데이터, 메타데이터로 구성

일부분만 수정하는 것이 불가능
방대한 확장성 그리고 메타 데이터
규모와 유연성이 필요한 애플리케이션을 구축하는데 적합

** 메타데이터(Metadata)는 데이터를 설명하는 데이터입니다. 즉, 어떤 데이터에 대한 추가적인 정보(설명, 속성, 구조 등)를 제공하는 데이터
예시)
(1) 파일 메타데이터 (운영체제 파일 시스템)
파일 크기: 100MB
파일 형식: JPEG, PNG, TXT 등
생성 날짜: 2025-02-05
수정 날짜: 2025-02-06
접근 권한: 읽기/쓰기 가능 여부
저장 위치: /home/user/docs/report.pdf

파일 스토리지(File Storage)
파일 단위로 저장하는 스토리지
파일시스템을 통해 데이터를 파일 단위로 저장
파일의 일부분만 수정이 가능

블록 스토리지(Block Storage)
블록 단위로 저장하는 스토리지
각 블록은 고유한 주소 또는 식별자를 가지고 있다. 블록 저장공간을 나누는 단위
파일 시스템이나 메타데이터 없이 순수한 데이터 블록만을 저장.

엄청나게 낮은 지연 시간을 제공.
고성능 워크로드에 적합
주로 데이터베이스, 가상 머신, 클라우드 스토리지 등에 저장

파일 시스템을 설치하면 파일 스토리지랑 비슷해짐.

파일 스토리지 vs 객체 스토리지

파일 스토리지                               객체 스토리지
데이터를 파일 단위로 관리                데이터를 객체 단위로 관리
파일 내용 일부분만 수정 가능             객체 데이터 일부분만 수정 불가능
운영체제에서 직접 접근 가능(볼륨 마운트)    운영체제에서 직접 접근 불가
큰 확장에 대처하기 어려움                큰 확장에 유연하게 대처 가능
사본을 저장하지 않음                    다수의 사본을 분산시켜 저장(안정성)

파일 스토리지에서는 파일을 찾기 위해 파일의 경로를 알아야 함.
객체 스토리지에서는 각 객체가 고유한 아이디를 갖고 있기 때문에 해시 함수를 통해 곧바로 찾을 수 있음.

객체 스토리지는 깊이(depth)가 1인 tree라고 이해하면 됨.
